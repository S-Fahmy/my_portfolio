Attendances Tracking web app

The web app is part of an HR app i’ve worked on, It’s Responsible for importing employees attendances data from FP machine (currently it can integrate with multiple ZTeco machines).

MY ROLE:

I am the fullstack developer, My goal was to implement the requested custom features this app needs.

Tech Used: for the backend, it's built using the Flask framework, postgreSQL for the database, and the frontend is built with VueJS and Bootstrap .

these main custom features include options like where the responsible admin can manage custom schedules and rules, and assign them to all or specific employees.

And as the main goal of using this web app is to calculate and generate a detailed employees attendances report in between selected date range,

the detailed report contains each day employees abscenct status, Total overtime and undertime worked in comparison to the active schedule, Along with a total summary report.

Responsible Admin can register excused absences, early leaves and events with the option to sync data with google calendar.

CHALLENGES

One of the challenges for this project was figuring out and planning the right workflow to integrate my API with the FP machine API to import the needed data and reformat it in an optimized data structure other areas of this app can use.
